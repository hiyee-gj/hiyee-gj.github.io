001. Project - 초기세팅, mysqldump, django seed

project(greatingdor)

mysqldump

  • djagno에서 프로젝트를 하다보면 models.py에서 수정사항이 생기는 경우가 많다
  • models.py를 수정하고 적용하려고 보면, 데이터베이스가 꼬이거나 제대로 적용이 안되는 경우 또한 빈번히 발생한다
  • 더미 데이터의 경우 이 글 다음에 오는 django seed를 사용하면 되지만, 중요한 데이터의 경우 저장해 놓는 편이 안전하고, 또 저장한 데이터를 불러와서 데이터베이스를 다시 구성한다면, 복구하는데 걸리는 시간 또한 적게 들어간다
  • mysqldump는 MySQL의 대표적인 백업 프로그램으로 스토리지 엔진에 상관없이 백업을 받을 수 있는 도구이다(물리적인 데이터 복사가 아닌 Logical 백업 도구임), 내가 저장하려는 데이터베이스를 sql 형태로 저장하는 도구이다
  • 사용법에 대해서는 더 다양한 사용법이 존재하지만, 여기서는 전체 데이터베이스를 저장하고, 다시 복구하는 사용법만 기록한다, 테이블 단위나, 더 세세한 적용법에 대해서는 구글링을 해보길 바란다

Tip

  1. 데이터가 저장되길 원하는 장소에서 명령문을 실행해도 되고, 경로를 지정할 수도 있지만 주로 원하는 곳에서 실행하는 것 같다
  2. 명령어를 입력할 때, 어려운 부분은 없지만 괄호의 방향을 헷갈리면, 데이터베이스를 손상시킬 수 있기 때문에 신경써 주는 것이 좋다
Export:

1. 데이터베이스 전체를 추출할 경우
mysqldump -u 아이디 -p -A > 파일이름.sql

2. 특정 DB를 백업하는 경우(주로 사용)
mysqldump -u 아이디 -p DB명 > 파일이름.sql

Import:

1. mysql -u root -p < 파일이름.sql

  • 데이터를 지우기 전의 dogs의 모습
  • sql 파일을 저장할 위치로 이동한 모습
  • 원하는 DB를 저장하는 모습
  • DB를 다시 복구하는 과정
  • 여기서 삭제한 dogs를 데이터베이스에 다시 생성해 논 상태이다(데이터베이스에는 새롭게 생성된 dogs가 비어있는 상태로 존재하고 있음)

  • 다시 복구된 데이터 모습
  • 내부의 테이블과 데이터까지 모두 살아있는 모습

django seed

  • 프로젝트를 하다보면 백엔드에서 테스트를 진행할 더미 데이터가 필요하게 된다
  • 직접 입력해서 데이터를 구성하는 것도 좋지만, 위에서 mysqldump에서 설명한 경우가 발생할 때마다 또다시 데이터를 쌓는 작업은 피로하고, 비효율적인 작업처럼 느껴진다
  • 이럴 경우 활용 가능한 것이 바로 Django Seed와 Faker이다
    • Faker라는 라이브러리 안에 Django seed가 존재하기 때문에, django seed만 설치한다면 faker 또한 사용할 수 있다

  • 또한 각 모델에 따라 OneToOneField냐, ManyToManyField냐, ForeignKeyField냐에 따라 작성하는 코드가 조금씩 달라진다(FK에 관한 문제인데, 이 내용은 구글링을 통해서 찾아서 해결하길 바란다, 이 글은 기본적인 기능에 대한 정리)

    설치 pip install django-seed

  • django seed는 기본적으로 앱의 models.py의 필드를 보고 생성해주어야 할 더미 데이터에 대해서 판단한다
    • 예를 들면 Emailfield의 경우 Charfield와 크게 다를 점이 없어서 사용하지 않았지만, django seed는 EmailField를 email로 인식해서 email 형식으로 출력해 준다
  • 때문에 Seed 파일을 생성할 때는 내가 적용하고자 하는 앱의 하위에 생성해 줘야 한다

    원하는 앱 안으로 이동 후 mkdir management cd management mkdir commands touch init.py commands/init.py commands/seed_users.py

  • 적용하고자 하는 앱 users 안에 management와 commands 생성한 모습
  • 작성한 seed의 내용이다, import로 Seed와 Faker를 불러왔다
  • 위의 argument 부분은 옵션의 기능이라고 생각하면 된다(생략해도 되는 부분)
    • 읽어보면 --help에 대한 정의 부분인 듯
  • handle 부분이 자동 생성할 데이터에 대한 설정 부분이다
    • handle을 살펴보면, name, phone_number, password 에 대해서 lambda를 이용해서 데이터 값의 형태를 잡아준 모습이다
    • 위의 number 부터 seeder 부분은 설정해줘야 하는 값들이고, seeder 안의 locale 부분은 내가 name에서 한국 이름을 생성하고 싶어서 설정한 값이다(faker 또한 같은 맥락)
  • 여기서 보면 email에 대한 설정은 없다
  • users/models.py에서 EmailField를 사용한 모습
  • django seed가 자동으로 생성해준 이메일, 이처럼 django seed가 이메일 필드를 인식하고 생성해줬기 때문에, 앱 안에 seed를 작성해야 한다는 점에서는 이해가 됐을 거라 생각한다
  • 우리가 원하는 앱 안에 원하는 형태로 seed를 작성하게 되면, manage.py가 있는 곳에서 --help를 찍었을 때, seed 기능이 생성된다
  • 그리고 manage.py가 있는 곳에서 명령어를 실행하면 된다

    python manage.py seed_users(생성된 seed파일 이름)

  • 성공했을 경우 몇 개를 생성했는지 성공 메시지를 알려준다(seed 파일에서 설정한 내용임)

seed로 데이터 형태 다듬기

  • 여러 데이터 형태를 확인하기 위해 테스트용 app을 만들어서 확인한다
  • 모델
  • 시드파일

seed 응용

  • 관계가 형성된 model을 자동 생성해주면 생기는 문제가 FK에 대한 문제가 발생하는 경우다
  • 이 경우에는 관계가 맺어진 테이블을 불러온 후 FK에 불러온 테이블의 ID를 랜덤하게 설정해주는 방법으로 해결한다
  • 이 경우를 응용해서 url 필드의 내용을 실제로 존재하는 img url을 저장한 파일을 불러와 랜덤하게 설정해서 실제 작동되도록 하는 방법을 공부할 예정이다
