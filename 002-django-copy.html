002. Django - tutorial 2 review

tutorial 2 review

  • django는 기본적으로 SQLite를 제공하고 있다 하지만 나는 MySQL을 사용하기 위해 설정을 조금 변경하는 과정을 소개한다

MySQL에서 DATABASE 생성하기

  1. mysql.server start로 서버를 열어주기
  2. mysql -u root -p를 통해 mysql 접속
  3. DATABASE 생성 testdata는 데이터베이스 이름임
  4. SHOW DATABASES를 통해서 생성됐는지 확인

mysite/settings.py에서 MySQL로 변경해주기

INSTALLED_APPS

  • mysite/settings.py에 확인하면 installed_apps에 django에서 기본으로 제공하는 앱들을 확인할 수 있다
  • 앱들의 기능에 대한 내용이다
  • 기본 어플리케이션들 중 몇몇은 최소한 하나 이상의 데이터베이스 테이블을 사용한다
  • 그러기 위해선 데이터베이스에서 테이블을 미리 만들 필요가 있음
  • 다음 명령을 통해 테이블 생성 python manage.py migrate

model 작성하기

  • Model이란 부가적인 메타데이터를 가진 데이터베이스의 구조를 말한다
  • Question과 Choice라는 모델을 작성
  • question 에는 질문(문자 최대길이 200) 내용 생성 날짜(시간 타입)
  • choice에는 선택지에 해당하는 질문(외래키=question의 내용을 참조하겠다, CASCADE = question이 삭제가 되면 choice question도 삭제 하겠다), 투표수의 내용이 담겨 있음
  • 여기서 외래키에 대해서 더 살펴보면 하나의 Question에 여러개의 foreignkey를 갖기 때문에 1대 다 관계인 것을 확인할 수 있다

INSTALLED_APPS에 polls.app 추가하기

makemigration

  • makemigration : polls에 model을 작성해 놓은 것을 mygration이라는 장소에 이 model들을 데이터베이스 내에 테이블을 생성할 수 있도록 설계도를 만드는 작업
  • 실행 결과(migration 디렉토리 안에 생성된 것을 확인할수 있음)

migrate

  • migrate을 통해 실제 테이블 생성

API 가지고 놀기

  • API란 개발자가 필요로하는 데이터를 뽑아낼 수 있도록 만들어 놓은 함수 또는 서버에게 데이터베이스에게 데이터를 입력할 수 있도록 만들어 놓은 함수
  • shell 진입
  • shell에 만들어 놓은 model 추가
  • Question의 모든 내용 불러오기
  • Question.object.all() - 현재 퀘스쳔 내의 모든 데이터를 가져와라
  • 등록된 내용 없기 때문에 빈 내용 출력
  • 다음으로 Question 내의 데이터 생성
  • Question 내에는 질문내용과 발행일이 존재
  • 발행일 시간을 위해 timezone 라이브러리 import
  • q.id 는 함수에서 명시하지 않아도 장고에서 추가 해줌
  • Question의 모든 내용 불러오기
  • object에 표시된 1로는 어떤 데이터인지 구분이 안감
  • 그렇기 때문에 __str__() 메소드를 모델 내에 심어주면 보고 싶은 문구를 볼 수 있음
  • model안에 __str__() 심어주기
  • 저장하고 shell을 종료시킨 후 재실행해서 Question 모든 내용을 불러오면
  • str 결과를 볼 수 있다
  • 추후 여러 실행결과를 리뷰할 예정
