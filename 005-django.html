005. Django - model method, 정참조, 역참조

Queryset

  • Queryset은 전달받은 Model의 객체 목록을 의미한다
  • Database로부터 데이터를 읽고, 필터를 걸거나 정렬 등을 할 수 있다
  • 리스트와 같은 구조를 지니고 있지만, 파이썬 기본 자료구조가 아니기 때문에 읽고, 쓰기 위해서는 자료형 변환을 해줘야 한다
  • Queryset은 Database의 여러 레코드(row)를 나타낸다

objects

  • 여기서 보이는 objects는 ModelManager로 일종의 매니저 역할을 한다고 생각하면 된다
  • Database와 django 사이에서 의사소통을 할 수 있도록 돕는 역할을 한다
  • Product.objects를 보면 objects라는 이름의 매니저가 Product의 DB를 Queryset 형태로 변환시킨다는 뜻이다

Model method

Methods That Don’t Return QuerySets

  • create()
    • Table에 데이터를 추가(INSERT) 해주는 method로, 생성된 인스턴스를 반환해준다
  • get()
    • 지정된 조회 매개 변수와 일치하는 인스턴스를 반환합니다
    • 매개 변수는 필드 조회에 설명된 형식이어야합니다
    • 하나의 값만 가능!! 둘 이상이라면 오류 발생
  • update()
    • 지정된 필드에 대해 업데이트 쿼리를 수행하고 수행한 행 수를 반환
    • 일부 행에 이미 업데이트 하려던 값이있는 경우 업데이트 된 행 수와 일치하지 않을 수 있다
    • update()를 사용하려면 queryset 형태에만 적용할 수 있다
  • delete()
    • QuerySet의 모든 행에 대해 SQL 삭제 쿼리를 수행하고 삭제 된 개체 수와 개체 유형별 삭제 횟수가 있는 dictionary를 반환한다
  • save()
    • 저장
  • exists()
    • filter()와 함께 사용해서 filter 조건에 맞는 데이터가 있는지 조회, 존재하면 True 존재하지 않으면 False를 반환한다
  • get_or_create(), bulk_create(), count(), first(), last(), aggregate() 등등(사용해보고 리뷰 올리겠습니다!!)

Methods That Return QuerySets

  • all()
    • 한 테이블의 모든 레코드를 가져오려면 all() method를 사용한다
    • 그 결과로 QuerySet 을 반환한다
    • 이때, QuerySet 안에는 각각 인스턴스가 포함되어 있다
  • exclude()
    • 조건에 맞지 않는 값을 조회
    • IS NOT, NOT IN
  • filter()
    • 한 테이블의 특정 레코드를 가져올 때 사용
    • filter() method는 가장 자주 사용하는 기능
    • filter(**kwargs) : 키워드 인자로 주어진 조건에 일치하는 레코드들의 QuerySet을 반환한다
  • values()
    • 쿼리셋의 값을 딕셔너리 형태로 반환한다
  • values_list()
    • 쿼리셋의 값을 튜플 형태로 반환한다

ManytoMany field

Foreignkey Field로 이어주는 것과 차이점

  • 중간 테이블 AllergyDrink를 만들어주고 Allergy와 Product를 FK로 연결해줘도 사실 문제는 없다
  • 그런데 굳이 ManytoMany field로 설정해주는 이유는 뭘까?
  1. 데이터를 삽입할 때 차이 (create)
  • Product와 Allergy를 ManytoMany로 연결해준 경우에는 굳이 ` AllergyDrink를 거치지 않고 Product를 통해 바로 Allery`에 접근 할 수 있다
  • 만약 AllergyDrink에서 FK를 통해 Product와 Allergy를 연결해주면 데이터 추가할때 항상 중간테이블을 거쳐야한다
  1. 데이터를 불러올 때 차이 (get)
  • Many to Many로 연결한 경우 Product에서 Allergy 정보(정참조)를 바로 가져올 수도 있고, Allergy에서도 Product정보(역참조, 장고에서 붙여준 매니저 _set을 통해)를 바로 가져올 수 있다

정참조

In [2]: d1 = Product.objects.get(id=1)

In [3]: d1.category.name

Out[6]: <QuerySet [{'id': 2, 'name': '우유'}]>

Out[7]: '우유'

역참조

  • related name을 설정해주지 않았으면 _set이 붙으면 역참조 데이터임을 알 수 있다
  • 하지만 함부로 설정했을 경우, 역참조를 도와주는 매니저인지 모를 경우가 있을 수도 있으니, 초창기에는 _set인 상황에서 익숙해지는 걸 추천
