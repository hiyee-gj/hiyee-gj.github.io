006. Django - C.R.U.D 2 review

C.R.U.D 2 review

  • C.R.U.D 1 때 models.py를 작성하고, mysql에 migrate해서 django shell을 이용해 데이터를 입력하고, 삭제하고, 업데이트 하는 등 조작을 해 봤다
  • C.R.U.D 2 에서는 프로젝트 urls.py와 앱 urls.py를 작성하고, views.py를 작성해서 httpie를 통해 request 온 내용을 처리하는 과정을 학습한다

flow

  • review에 앞서 흐름에 대해 이해하고 넘어가려한다
  • 위 그림에서 client가 request를 보내면 프로젝트 urls.py에서 분류하여 앱의 urls.py로 보내준다, 앱 urls.py는 여기서 한번 더 분류해서 일치하는 views.py의 class로 보내주게 되고, views.py는 해당 로직을 완수한다

models.py

  • 환경설정과 기초 세팅에 관해서는 생략하고 모델부터 살펴보겠다
  • Owner와 Dog클래스가 있고, 1대다 관계를 형성하고 있는 모델이다

views.py

post

  • 먼저 POST 메소드를 처리할 부분을 살펴보겠다
  • 두 클래스의 메소드 이름이 모두 post인 것에 집중해야 한다
  • 프로젝트 urls.py에서 앱 urls.py로 온 후에 각 뷰에 as_view()를 적용해 주었다, as_view() 부분이 request에 담긴 메소드를 따라서 views.py안의 클래스 안에 있는 여러 메소드 중 일치하는 메소드의 로직을 따르도록 하는 부분이다

get

  • 다음은 get 메소드를 처리할 부분이다
  • Owner에서 get을 처리할 때, 한번에 정보를 출력하기 위해서 코드를 설계하다보니 많이 지저분한 모습이다
  • 이후 C.R.U.D 2 세션에서 더 깔끔한 모습을 봤으니 다음 westagram 과제에서 사용하게 된다면 더 깔끔하게 구현하도록 노력해야겠다

프로젝트 urls.py

  • ownerdogs 부분이 request로 들어오면 include안의 앱의 urls.py로 보내게 처리한 내용이다

앱 urls.py

  • 프로젝트 urls.py에서 분류되어 들어와서 /owner 나 /dog로 분류되어 원하는 클래스로 분류하는 부분이다
