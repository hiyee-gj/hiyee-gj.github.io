007. Django - westagram review 1

Westagram

초기 세팅

  • conda 새로운 가상환경 westagram 생성
  • pip install django, mysqlclient, ipython, django-cors-headers 설치
  • github에서 westagram 클론해오고, 가이드 따라서 프로젝트 생성
  • my_settings 생성해서 github에 올라가면 안되는 설정들 옮겨놓고, import
  • settings.py에서 주석처리할 내용들 주석처리하고, installed.apps에 pip install 했던 corsheaders와 middleware 설정
  • 프로젝트에 users 앱 생성, settings.py에서 users 앱 추가

models.py 작성

models.py review

  • email에서 EmailField가 특별한 기능을 가지지 않아 CharField 써도 똑같을 것 같음
  • phone에서 PhoneField를 사용하기 위해서 pip install 까지 했는데 PhoneField가 하는 역할이 없어서 사용할 필요성이 없어 보임

validation.py 작성

  • password, email, phone의 유효성을 확인하기 위해서 validator 작성
  • 정규표현식으로 작성하면 더 간단하게 처리할 수 있음

views.py 작성(회원가입)

회원가입 review

  • validation.py 적용하기 전 모습
  1. 파이썬 컨벤션에 맞게 =에 맞춰 정리
  2. 성공했을 경우 return 값이 해당 코드 바로 뒤에 오게 해서 가독성 높이기
  3. if문 return 하는데 굳이 elif 쓰는 이유
  4. 오류 메시지는 프론트엔드 개발자가 보는 내용이기 때문에 굳이 한글로 쓸 필요 없음
  5. if문 사이 빈칸 추가로 가독성 높이기
  6. number 변수명 더 명확하게 수정하기
  7. class명 회원가입의 경우 SignUpView를 사용하는게 일반적

변경된 views.py(회원가입)

  • 추가 변경사항 - validation이 먼저 완료되고, 데이터베이스에 접근하도록 수정함
