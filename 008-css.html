008. CSS - position, inline&block 되새김과 float

Position

  • 참고 자료(생활코딩)

static

  • position을 주지 않았을 때 기본값
  • left, right, top, bottom 에 값을 줘도 변화하지 않는다(static 일 경우 비활성화 상태이기 때문)

relative

  • 원래 자기가 있던 위치를 기준으로 움직임
  • 부모 태그에 position: relative 를 설정하고, 부모의 위치를 움직이면 자신의 위치도 그에 맞게 움직임
  • 원래 자신의 위치를 기억하는 특징을 가진다. 예를 들어 자식 태그의 relative의 top의 값을 0을 주면, 자신의 원래 위치였던 top을 기준으로 돌아간다
  • relative는 무조건 자신의 부모 태그에만 연관되어 움직임

absolute

  • absolute은 설정되면 부모 태그 안에서 공간적 자유를 얻게 된다(부모 태그에는 position: relative가 설정되어 있어야 함)
  • left, right, top, bottom의 값이 부모 태그를 기준으로 움직이는 점은 얼핏 relative의 특성과 비슷해 보이지만 차이점이 존재한다 1. relative는 부모의 위치와 관련성이 있지만 자신의 원래 위치와도 큰 연관성을 가지고 움직인다 (위의 설명처럼 top 또는 방향의 값이 0이면 본래 자신의 위치로 복귀한다) 2. absolute는 본인의 위치에서는 완전한 자유를 얻는다
  • 본래 absolute 자리에 fixed 요소가 올라와서 자리를 차지하는 모습을 볼 수 있다
  • absolute의 top 값에 0을 주자 부모 태그의 모서리의 붙는 모습
  • absolute는 자신의 부모 태그들 중 position이 static 상태가 아닌 부모 태그에 연관되어 움직임(때문에 기준이 될 부모 태그를 잘 설정해야 함)

fixed

  • absolute와 비슷하게 설정되었을 때 자신의 본래 위치에서 자유로운 모습을 보여준다(파란부분(fixed))
  • fixed는 브라우저 view 기준으로 움직인다
  • top 값을 0으로 설정했을 때, 화면 처음으로 간 모습
  • 가장 큰 특징은 스크롤을 해도 그 위치를 그대로 유지하는 모습이다

inline & block

  • 참고블로그 01 참고블로그 02

inline

  • 위의 사진에서 가장 좌측의 inline을 보면 content만큼의 크기를 가진 것을 볼 수 있다
  • 또 줄바꿈이 없어 바로 옆에 다음 요소가 위치해 있는 것을 알 수 있다
  • width와 height를 설정할 수 없다
  • margin 값을 좌우로는 설정할 수 있지만 상하로는 설정할 수 없다
  • a, span 태그 등이 inline의 특성을 지니고 있다

block

  • block은 한 영역을 차지 하는 박스형태를 가지고 있다
  • 때문에 기본적으로 block은 width값이 100%라고 생각하면 된다
  • 줄바꿈이 적용되어 다음 요소를 다른 줄로 밀어낸다
  • block은 height와 width 값을 설정할 수 있다
  • block은 margin과 padding을 설정 할 수 있다
  • div, p, h1 태그 등이 block 특성을 가지고 있다

iline-block

  • inline과 block의 특성을 모두 가진 속성이다
  • height와 width 값을 설정할 수 있다
  • margin과 padding의 상하 값을 설정할 수 있다
  • 줄바꿈이 이뤄지지 않는다
  • 만약 width와 height를 지정하지 않을 경우, inline과 같이 컨텐츠만큼 영역이 잡힌다.

float(참고자료(생활코딩))

  • 본문에 이미지를 삽입할 때, 이미지를 자연스럽게 배치하기 위해서 사용한다
  • 레이아웃을 잡을 때에도 많이 사용한다
  • flex를 주로 사용하게 되면서 사용빈도는 낮아졌지만, 예전 방식으로 만들어진 웹페이지에는 종종 등장하기 때문에 공부해 두면 좋다
  • 위 그림을 보면 본문의 오른편에 이미지가 들어가 있고, 본문의 내용이 이미지를 피해 자동으로 줄바꿈을 하고 있는 것을 볼 수 있다
  • 본문 중 아래 본문은 이미지와 조화를 이루는 것이 아닌 다음으로 넘기고 싶어
  • 두번 째 본문 내용을 담고 있는 p 태그에 속성 style의 clear:both(이미지가 어느쪽에 있던지 적용가능)로 float를 초기화 시켜준 모습이다
  • 지금같은 경우는 이미지가 왼편에 있기 때문에 clear:left로 해도 동일한 결과가 도출된다
  • float를 초기화 시키는 방법은 4가지로 소개되고 있지만 비추천하는 방법들이 대부분이기 때문에 위의 방법과 부모태그를 css에서 display:inline-block으로 설정하는 방법 두 가지를 추천한다
